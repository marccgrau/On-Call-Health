2. **TokenManager class:**
   ```python
   class TokenManager:
       """Abstraction layer for retrieving valid tokens from integrations.

       Provides get_valid_token() that handles:
       - OAuth tokens: checks expiry, refreshes if needed, returns decrypted token
       - Manual tokens: returns decrypted token directly (no refresh, no validation)

       Callers don't need to know the token source - this service handles it.

       Note: Manual token validation (checking if token still works) is Phase 2 scope.
       This class only handles token retrieval and OAuth refresh.
       """

       def __init__(self, db: Session):
           self.db = db
   ```

3. **Main method `async def get_valid_token()`:**
   - **IMPORTANT: This method is async** because OAuth refresh calls async APIs
   - Accepts `integration: Union[JiraIntegration, LinearIntegration]`
   - Returns `str` (decrypted token)
   - Raises `ValueError` if no token available or refresh fails
   - Routes to `_get_oauth_token()` if `integration.is_oauth`
   - Routes to `_get_manual_token()` if `integration.is_manual`
   - Raises `ValueError` for unknown token_source

4. **Private method `async def _get_oauth_token()`:**
   - Check `integration.has_token`, raise ValueError if missing
   - Refresh integration from DB: `self.db.refresh(integration)`
   - Check if refresh needed via `needs_refresh(integration.token_expires_at)`
   - If no refresh needed, return `decrypt_token(integration.access_token)`
   - If refresh needed but `not integration.supports_refresh`, raise ValueError
   - If refresh needed, call `refresh_token_with_lock()` with:
     - provider: `"jira"` if `isinstance(integration, JiraIntegration)` else `"linear"`
     - integration_id: `integration.id`
     - user_id: `integration.user_id`
     - refresh_func: Lambda that calls `self._perform_oauth_refresh(integration)`
     - fallback_func: Lambda that calls `self._perform_oauth_refresh_with_db_lock(integration)`

5. **Private method `async def _perform_oauth_refresh()`:**
   - Pattern extracted from `IntegrationValidator._perform_jira_token_refresh()` and `_perform_linear_token_refresh()`
   - Double-check if refresh still needed after acquiring lock
   - Decrypt refresh_token
   - Call appropriate OAuth API based on integration type:
     - Jira: POST to `https://auth.atlassian.com/oauth/token`
     - Linear: POST to `https://api.linear.app/oauth/token`
   - Update integration with new tokens (encrypted)
   - Commit to DB
   - Return decrypted new access_token

6. **Private method `async def _perform_oauth_refresh_with_db_lock()`:**
   - Pattern extracted from `IntegrationValidator._perform_*_token_refresh_with_db_lock()`
   - Use database row locking as fallback when Redis unavailable
   - Same logic as _perform_oauth_refresh but with FOR UPDATE lock

7. **Private method `_get_manual_token()`:**
   - Check `integration.has_token`, raise ValueError if missing
   - Return `decrypt_token(integration.access_token)` directly
   - **No validation here** - manual tokens are assumed valid until API call fails
   - Validation/caching for manual tokens is Phase 2 scope

8. **Logging:**
   - Log at INFO level when refresh is needed
   - Log at DEBUG level for token retrieval
   - Never log the actual token value

**Relationship to existing code:**
- TokenManager is a NEW abstraction layer that sits above IntegrationValidator
- It DELEGATES to existing utilities: `decrypt_token`, `needs_refresh`, `refresh_token_with_lock`
- It EXTRACTS the OAuth refresh logic pattern from IntegrationValidator's `_get_valid_jira_token` and `_get_valid_linear_token`
- IntegrationValidator's existing methods continue to work (no changes to IntegrationValidator in this plan)
- Future work may migrate IntegrationValidator to use TokenManager internally

**Error messages should be user-friendly:** "No access token available" not "token is None"
**Match error message patterns from IntegrationValidator:** "Authentication error. Please reconnect {Provider}."
