Output: New `backend/app/services/token_manager.py` with TokenManager class.

2. **TokenManager class:**
   ```python
   class TokenManager:
       """Abstraction layer for retrieving valid tokens from integrations.

       Provides get_valid_token() that handles:
       - OAuth tokens: checks expiry, refreshes if needed, returns decrypted token
       - Manual tokens: returns decrypted token directly (no refresh possible)

       Callers don't need to know the token source - this service handles it.
       """

       def __init__(self, db: Session):
           self.db = db
   ```

3. **Main method `get_valid_token()`:**
   - Accepts `integration: Union[JiraIntegration, LinearIntegration]`
   - Returns `str` (decrypted token)
   - Raises `ValueError` if no token available or refresh fails
   - Routes to `_get_oauth_token()` if `integration.is_oauth`
   - Routes to `_get_manual_token()` if `integration.is_manual`
   - Raises `ValueError` for unknown token_source

4. **Private method `_get_oauth_token()`:**
   - Extract logic pattern from `IntegrationValidator._get_valid_jira_token()` and `_get_valid_linear_token()`
   - Check `integration.has_token`, raise ValueError if missing
   - Check if refresh needed via `needs_refresh(integration.token_expires_at)`
   - If no refresh needed, return `decrypt_token(integration.access_token)`
   - If refresh needed but no refresh_token, raise ValueError
   - If refresh needed, call `refresh_token_with_lock()` with appropriate provider
   - Provider determined by checking type: `isinstance(integration, JiraIntegration)` -> "jira", else "linear"

5. **Private method `_get_manual_token()`:**
   - Check `integration.has_token`, raise ValueError if missing
   - Return `decrypt_token(integration.access_token)` directly
   - Manual tokens don't expire server-side (validation happens elsewhere)

6. **Logging:**
   - Log at INFO level when refresh is needed
   - Log at DEBUG level for token retrieval
   - Never log the actual token value

**Important patterns to follow:**
- Use existing `decrypt_token()` from integration_validator - do NOT duplicate encryption logic
- Use existing `refresh_token_with_lock()` from token_refresh_coordinator for OAuth refresh
- Error messages should be user-friendly: "No access token available" not "token is None"
- Match error message patterns from IntegrationValidator ("Authentication error. Please reconnect {Provider}.")
