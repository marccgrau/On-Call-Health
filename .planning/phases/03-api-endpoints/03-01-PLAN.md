  <action>
Create `backend/app/api/endpoints/api_keys.py` with CRUD endpoints following the existing patterns in `invitations.py`.

**Request model (CreateApiKeyRequest):**
- `name`: str, required, min_length=1, max_length=100
- `expires_at`: Optional[datetime], must be in future if provided
- Use Pydantic v2 `@field_validator` with `@classmethod` decorator (NOT v1 `@validator`)
- Strip whitespace from name, reject empty-after-strip

**Endpoints:**

1. `POST /api-keys` (status_code=201)
   - Use `@integration_rate_limit("integration_create")` decorator
   - Dependency: `current_user: User = Depends(get_current_active_user)` (JWT only)
   - Dependency: `db: Session = Depends(get_db)`
   - Call `APIKeyService(db).create_key(user_id=current_user.id, name=request.name, expires_at=request.expires_at)`
   - Catch `ValueError` and return HTTP 400 with detail message
   - Return flat JSON (per CONTEXT.md): `{"id": ..., "name": ..., "key": full_key, "masked_key": ..., "scope": ..., "created_at": ..., "expires_at": ...}`
   - Use `.isoformat()` for datetime fields, handle None values

2. `GET /api-keys` (status_code=200)
   - Use `@integration_rate_limit("integration_get")` decorator
   - Same dependencies as create
   - Call `service.list_user_keys(user_id=current_user.id, include_revoked=False)`
   - Return wrapped JSON (per CONTEXT.md): `{"keys": [...]}`
   - Each key object: `{"id": ..., "name": ..., "masked_key": ..., "scope": ..., "is_active": ..., "created_at": ..., "last_used_at": ..., "expires_at": ...}`
   - NEVER include `key` field in list response (security)

3. `DELETE /api-keys/{key_id}` (status_code=204)
   - Use `@integration_rate_limit("integration_update")` decorator
   - Path parameter: `key_id: int`
   - Same dependencies as create
   - Call `service.revoke_key(key_id=key_id, user_id=current_user.id)`
   - If returns False, raise HTTP 404 with detail "API key not found or already revoked"
   - Return `Response(status_code=status.HTTP_204_NO_CONTENT)`

**Router setup:**
```python
router = APIRouter(
    prefix="/api-keys",
    tags=["api-keys"]
)
```

**Imports needed:**
- `from datetime import datetime, timezone`
- `from typing import Dict, Any, Optional`
- `from fastapi import APIRouter, Depends, HTTPException, status, Response`
- `from sqlalchemy.orm import Session`
- `from pydantic import BaseModel, Field, field_validator`
- `from ...models import get_db, User`
- `from ...auth.dependencies import get_current_active_user`
- `from ...services.api_key_service import APIKeyService`
- `from ...core.rate_limiting import integration_rate_limit`

**DO NOT:**
- Import `get_current_user_from_api_key` or any API key auth (JWT only per CONTEXT.md)
- Return full key in list response
- Use Pydantic v1 `@validator` decorator
