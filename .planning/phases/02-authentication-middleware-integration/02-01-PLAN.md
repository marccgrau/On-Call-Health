Create new file `backend/app/auth/api_key_auth.py` with:

1. **Imports:** FastAPI (Depends, HTTPException, status, Request, BackgroundTasks), fastapi.security.APIKeyHeader, sqlalchemy.orm.Session, asyncio, logging, datetime

2. **api_key_header scheme:**
   - Use `APIKeyHeader(name="X-API-Key", auto_error=False, description="API key for programmatic access")`
   - auto_error=False so we can provide custom error messages

3. **_update_last_used_background(api_key_id: int) function:**
   - Background task to update last_used_at timestamp
   - Create new session from SessionLocal factory (NOT passed session - thread safety)
   - Use raw SQL UPDATE for efficiency: `UPDATE api_keys SET last_used_at = :now WHERE id = :id`
   - Handle exceptions gracefully (log warning, don't fail)
   - Always close session in finally block

4. **get_current_user_from_api_key async dependency:**
   - Parameters: request: Request, background_tasks: BackgroundTasks, api_key: str | None = Depends(api_key_header), db: Session = Depends(get_db)
   - Returns: User

   Validation logic (IN THIS ORDER):
   a. Check for JWT in Authorization header - if starts with "Bearer " and doesn't start with "Bearer och_live_", raise HTTPException 400: "This endpoint requires API key authentication. Use X-API-Key header instead of Bearer token."
   b. If api_key is None, raise HTTPException 401: "Missing API key. Provide X-API-Key header." with headers={"WWW-Authenticate": "APIKey"}
   c. If api_key doesn't start with "och_live_", raise HTTPException 401: "Invalid API key format. Keys should start with 'och_live_'."
   d. Phase 1 - SHA-256 lookup: compute_sha256_hash(api_key), query APIKey by key_hash_sha256
   e. If not found, log info "API key lookup failed", raise HTTPException 401: "Invalid API key"
   f. Check revoked_at is not None: log info "API key {id} rejected: revoked", raise HTTPException 401: "API key has been revoked"
   g. Check expires_at: if not None and datetime.now(timezone.utc) >= expires_at, format date as YYYY-MM-DD, log info "API key {id} rejected: expired on {date}", raise HTTPException 401: "API key expired on {date}"
   h. Phase 2 - Argon2 verification: await asyncio.to_thread(verify_api_key, api_key, api_key_model.key_hash_argon2)
   i. If verification fails, log warning "API key {id} failed Argon2 verification", raise HTTPException 401: "Invalid API key"
   j. Load user by api_key_model.user_id
   k. If user not found, log error "API key {id} has orphaned user_id", raise HTTPException 401: "API key owner not found"
   l. Store api_key_id in request.state: request.state.api_key_id = api_key_model.id
   m. Schedule background task: background_tasks.add_task(_update_last_used_background, api_key_model.id)
   n. Log info: "API key auth success: key_id={id} user_id={user.id} key_name='{name}'"
   o. Return user

All HTTPException with 401 should include headers={"WWW-Authenticate": "APIKey"} (except the 400 for JWT rejection).
