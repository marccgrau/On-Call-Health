  <name>Task 1: Add POST /connect-manual endpoint to jira.py</name>
  <files>backend/app/api/endpoints/jira.py</files>
  <action>
Add a new endpoint POST /connect-manual that:

1. Accepts JSON body with fields: token, site_url, user_info (optional, from frontend validation)
2. Re-validates the token using IntegrationValidator.validate_manual_token (from Phase 2)
   - If validation fails, return 400 with error details
   - NEVER trust frontend validation - always re-validate on backend
3. On successful validation:
   - Encrypt token using existing encrypt_token() function (Fernet)
   - Query for existing JiraIntegration for current_user.id
   - If exists: update access_token, token_source='manual', token_expires_at=None, jira_site_url
   - If not exists: create new JiraIntegration with token_source='manual', token_expires_at=None
   - Set jira_site_url to site_url with 'https://' prefix stripped
   - Commit to database
4. After commit, trigger background sync:
   - Import JiraUserSyncService
   - Call sync_jira_users(current_user) as background task
   - Use asyncio.create_task() for fire-and-forget execution (no need for APScheduler for immediate one-shot)
5. Return success response:
   ```python
   return {
       "success": True,
       "integration": {
           "id": integration.id,
           "token_source": "manual",
           "token_valid": True,
           "jira_site_url": integration.jira_site_url
       }
   }
   ```

Place the endpoint AFTER the existing /callback endpoint and BEFORE /status endpoint.

Important security notes:
- Log token validation attempts but NEVER log actual token values
- Use Bearer authentication (Jira PATs use Bearer, not Basic auth)
- Handle case where user already has OAuth integration - overwrite with manual token
