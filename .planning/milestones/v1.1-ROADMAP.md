# Milestone v1.1: MCP Distribution

**Status:** ✅ SHIPPED 2026-02-04
**Phases:** 5-12
**Total Plans:** 16

## Overview

Transform the MCP server from a direct-database-access implementation to a distributed architecture where MCP clients connect via SSE-hosted endpoint or PyPI package, both calling oncallhealth.ai REST APIs instead of querying the database directly. This enables zero-installation access via hosted SSE and self-hosted deployment via uvx. Production deployment uses AWS ECS/Fargate.
