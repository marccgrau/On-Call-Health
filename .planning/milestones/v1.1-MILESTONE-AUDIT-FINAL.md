## End-to-End Flow Verification

### Flow 1: SSE Connection to Hosted Endpoint
**Status:** ✅ COMPLETE

**Path:**
1. Client → http://13.218.46.36:8080/sse
2. RequestContextMiddleware → Sets request context for API key extraction
3. MCPInfrastructureMiddleware → Validates API key, checks connection limits
4. CORSMiddleware → Adds CORS headers
5. FastMCP http_app → Handles MCP protocol
6. MCP tool → Extracts API key from context (env var first, header fallback)
7. OnCallHealthClient → Calls oncallhealth.ai REST API
8. Response → Normalized and returned

### Flow 4: API Key Authentication
**Status:** ✅ COMPLETE

**Path:**
1. MCP tool called (any of 5 tools)
2. `get_api_key()` checks `os.getenv("ONCALLHEALTH_API_KEY")` first
3. If not set, falls back to `request.headers["X-API-Key"]`
4. OnCallHealthClient initialized with API key
5. Client injects `X-API-Key` header into all HTTP requests
6. oncallhealth.ai REST API receives authenticated request

**Verification:** Dual-priority pattern implemented in both backend and PyPI package
